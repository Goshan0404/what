# RAG-пайплайн для поиска статей Авито

Семантический поиск по базе справочных статей: по пользовательскому запросу находятся релевантные статьи из корпуса 793 документов.

## Структура проекта

```
├── articles.f                  # Feather: 793 статьи (article_id, title, body)
├── calibration.f               # Feather: 500 запросов с ground_truth для оценки
├── test.f                      # Feather: 500 запросов для финального submission
├── submission.csv              # Финальный ответ (query_id, top-10 article_id)
├── experiment_results.json     # Результаты перебора гиперпараметров
├── chroma_storage/             # Локальное хранилище ChromaDB
├── clean_documents_body.py     # Парсинг HTML → чистый текст
├── split_data.py               # Стратегии чанкинга
├── extract_embeddings.py       # Извлечение эмбеддингов
├── data.py                     # Загрузка данных
├── chromadb_collection.py      # Обёртка над ChromaDB
├── pipeline.py                 # Конвейер индексации
└── data_collection.ipynb       # Основной ноутбук: EDA, обучение, оценка, инференс
```

## Обработка HTML-статей

Тела статей хранятся в виде HTML. Кастомный парсер `RAGHtmlParser` (`clean_documents_body.py`) конвертирует их в чистый текст, сохраняя структуру:

- **Заголовки** (`h1`–`h6`) → Markdown-разметка (`# ... ######`)
- **Абзацы** (`p`) → текст с переносом строки
- **Списки** (`ul`, `ol`) → маркированные/нумерованные списки (`- item`, `1. item`)
- **Таблицы** (`table`) → строки вида `header: value; header: value`
- **Ссылки** (`a`) → текст ссылки (URL опционально)
- **Фактоиды** (блоки с class `factoid`) → `[ВНИМАНИЕ]`, `[СОВЕТ]`, `[ВАЖНО]`, `[ПРИМЕЧАНИЕ]`
- **Спойлеры** (`class="spoiler"`) → раскрываются в обычный текст
- Удаляются: `<script>`, `<style>`, `<noscript>`, `<iframe>`, `<svg>`, `<canvas>`, эмодзи
- Нормализуются: `&nbsp;`, множественные пробелы, более двух пустых строк подряд

Для TF-IDF-ветки применяется дополнительная обработка (`TFIDFPreprocessor`): lowercasing, удаление не-кириллических символов, стоп-слова, лемматизация через `pymorphy3`.


## Модели и алгоритмы

### Модель эмбеддингов
`ibm-granite/granite-embedding-311m-multilingual-r2` — мультиязычная модель для получения dense-векторов текста.

### Извлечение эмбеддингов
- **Mean pooling** по токенам с учётом `attention_mask` + L2-нормализация (`get_mean_embedding`)
- Максимальная длина последовательности: 512 токенов
- Также реализован CLS-токен как альтернатива (не использовался в лучшей конфигурации)

### Чанкинг
| Стратегия | Описание |
|---|---|
| `split_by_sentences` | Разбиение по предложениям (NLTK punkt) с лимитом `max_chars` |
| `split_with_overlap` | То же + перекрытие между чанками |
| `split_by_texttiling` | Тематическое разбиение (TextTilingTokenizer) |
| `simple_split` | Фиксированные куски по символам |

**Лучшая стратегия:** `split_by_sentences` с `max_chars=400`.

### Поиск и ранжирование
- **ChromaDB** — хранилище для векторных и TF-IDF эмбеддингов
- **TF-IDF** (`sklearn.TfidfVectorizer`) с кастомным русскоязычным препроцессором — альтернативная ветка поиска
- **Reciprocal Rank Fusion (RRF)** — объединение ranked-листов TF-IDF и семантического поиска (k=60)
- **Reranker** `BAAI/bge-reranker-v2-m3` — cross-encoder для финального переранжирования топ-20 кандидатов

## Оценка качества на calibration.f

Метрика: **MAP@10** (Mean Average Precision при k=10).

Для каждого из 500 запросов из `calibration.f` извлекается top-10 кандидатов и сравнивается с `ground_truth` (список релевантных article_id).


Основной эксперимент и инференс — в `data_collection.ipynb`.
